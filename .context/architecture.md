## 核心模块

### 1. AI 模块 (`src/lib/ai/`)

```
ai/
├── orchestrator.ts      # AI 对话编排器，协调各专业 Agent
├── classifier.ts        # 意图分类器，识别用户输入类型
├── conversation-flow.ts # 对话流程控制
├── memory.ts            # 记忆管理，存储对话上下文
├── emotional.ts         # 情感分析 Agent
├── persona.ts           # 人格画像 Agent
├── user-style.ts        # 用户风格分析 Agent
├── claude.ts            # Claude API 封装
├── embedding.ts         # OpenAI Embedding 封装
└── agents/              # 专业 Agent 实现
```

### 2. 数据层 (`src/lib/supabase/`)

- `client.ts` - 浏览器端 Supabase 客户端
- `server.ts` - 服务端 Supabase 客户端
- `admin.ts` - Service Role 客户端（绕过 RLS）

### 3. 后台任务 (`src/inngest/`)

- 事件驱动架构
- 支持重试和错误处理
- 定时任务（每日匹配）

## 数据流

### Onboarding 流程

```
用户输入 → Classifier → Orchestrator → 专业Agent → Memory → Profile
    │                        │
    │                        ├── EmotionalAgent (情感分析)
    │                        ├── PersonaAgent (人格画像)
    │                        └── UserStyleAgent (风格分析)
    │
    └── 实时反馈 (Streaming Response)
```


## 安全边界

```
┌─────────────────────────────────────────────────────────────┐
│                      公开访问                                │
│  • 登录页 • 注册页 • 静态资源                                │
└─────────────────────────────────────────────────────────────┘
                              │
                        Auth Middleware
                              │
┌─────────────────────────────────────────────────────────────┐
│                      认证后访问                              │
│  • Dashboard • Profile • Matches • Meetings                 │
│  • 所有 API Routes (除 auth callback)                       │
└─────────────────────────────────────────────────────────────┘
                              │
                         Supabase RLS
                              │
┌─────────────────────────────────────────────────────────────┐
│                      数据隔离                                │
│  • 用户只能访问自己的 Profile                                │
│  • 用户只能看到与自己相关的 Match                            │
│  • Meeting 数据按参与者隔离                                  │
└─────────────────────────────────────────────────────────────┘
```
